# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Fabric-Local** is a local Microsoft Fabric-like data lakehouse platform. It simulates a Fabric environment using PySpark 3.5, SQL Server 2022, and a Medallion Architecture (bronze/silver/gold Parquet data lake), all containerized with Docker and designed for VS Code Dev Containers on Windows WSL2.

## Environment Setup

The project runs inside a Docker Dev Container. Start it with:

```bash
docker compose up -d --build   # Build and start spark-dev + mssql containers
```

After first start, seed the source database:

```bash
python scripts/seed_mssql.py   # Creates fabric_demo DB with customers + orders tables
```

Environment variables are loaded from `.env` (copy from `.env.template` and fill in MSSQL credentials).

## Running the Pipeline

The data pipeline is orchestrated with **doit** (see `dodo.py`):

```bash
doit              # Run all notebooks in order: ingest → transform → aggregate
doit list         # Show available tasks
doit forget       # Reset task state to force re-run everything
```

Notebooks are executed headlessly via **Papermill** and outputs are saved to `notebooks/executed/`.

## Key Commands

| Command | Purpose |
|---------|---------|
| `doit` | Run full pipeline (01 → 02 → 03) |
| `python scripts/seed_mssql.py` | Seed MSSQL with demo data |
| `pre-commit run --all-files` | Run all pre-commit hooks |
| `docker compose logs mssql` | Check MSSQL container health |
| `docker compose down -v` | Stop containers and remove volumes |

Jupyter is available at port 8888, Spark UI at port 4040, MSSQL at port 1433.

## Architecture

```
MSSQL (fabric_demo DB)
    ↓  01_ingest.ipynb
data/lake/bronze/     (raw Parquet from MSSQL)
    ↓  02_transform.ipynb
data/lake/silver/     (cleaned, partitioned Parquet)
    ↓  03_aggregate.ipynb
data/lake/gold/       + write-back to MSSQL
```

### Key Files

- **`dodo.py`** — doit task definitions; defines the pipeline DAG with Papermill execution, task dependencies, and cleanup targets
- **`scripts/spark_session.py`** — Shared SparkSession factory and JDBC helpers used by all notebooks
- **`scripts/seed_mssql.py`** — Creates `fabric_demo` database with `customers` and `orders` tables
- **`conf/spark-defaults.conf`** — Spark configuration (2GB memory, Delta Lake catalog, MSSQL JDBC driver path)
- **`Dockerfile`** — Builds the container: Java 11, Spark 3.5.1, Python packages, MSSQL JDBC jar, Delta Lake jars, Claude Code CLI

### Available Jupyter Kernels

1. **PySpark 3.5** — Used by the 3 pipeline notebooks; supports `spark.sql()` and DataFrame API
2. **spylon-kernel** — Enables `%%sql` magic for Spark SQL in notebooks
3. **DuckDB** — For local SQL analytics without Spark overhead

### Notebook Git Hygiene

**nbstripout** is configured via `.gitattributes` to automatically strip notebook outputs on `git add`. The pre-commit hook also enforces this. Never commit notebooks with outputs — they'll be regenerated by `doit`.

## Delta Lake & JDBC

Delta Lake JARs (3.1.0) are pre-installed in the container. The MSSQL JDBC driver (12.4.2) is at `/opt/spark/jars/`. Both are referenced in `conf/spark-defaults.conf` and loaded automatically by the SparkSession in `scripts/spark_session.py`.
